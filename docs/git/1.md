## Git 入门教程

### 1. Git 是什么？

Git 是一个**分布式版本控制系统 (DVCS)**，它能高效地跟踪文件的修改历史，让多人协作开发变得简单有序。

### 2. 安装与配置

#### 2.1 安装 Git
- **Windows**: 从 [Git 官网](https://git-scm.com/download/win) 下载安装包
- **macOS**: 使用 Homebrew 安装 `brew install git`
- **Linux**: 使用包管理器安装 `sudo apt install git` (Debian/Ubuntu) 或 `sudo yum install git` (CentOS/RHEL)

#### 2.2 基本配置
安装完成后，首先需要配置你的身份信息：

```bash
# 配置用户名
git config --global user.name "Your Name"

# 配置邮箱
git config --global user.email "your.email@example.com"

# 查看配置信息
git config --list
```

### 3. Git 核心概念

- **工作区 (Working Directory)**: 你正在编辑的文件所在的目录
- **暂存区 (Staging Area/Index)**: 临时存放你的修改，准备提交
- **本地仓库 (Local Repository)**: 存储提交历史的数据库
- **远程仓库 (Remote Repository)**: 如 GitHub、GitLab 等服务器上的仓库

### 4. 基本操作流程

#### 4.1 创建/克隆仓库

```bash
# 初始化新仓库
git init

# 克隆远程仓库
git clone https://github.com/username/repository.git
```

#### 4.2 文件操作

```bash
# 查看文件状态
git status

# 将文件添加到暂存区
git add filename.txt          # 添加单个文件
git add .                     # 添加所有修改的文件

# 提交暂存区的修改
git commit -m "提交说明信息"

# 查看提交历史
git log                       # 查看详细历史
git log --oneline             # 简洁格式查看
```

#### 4.3 分支操作

```bash
# 查看分支
git branch                    # 查看本地分支
git branch -r                 # 查看远程分支

# 创建新分支
git branch feature-branch

# 切换分支
git checkout feature-branch

# 创建并切换分支 (推荐)
git checkout -b feature-branch

# 合并分支 (先切换到主分支)
git checkout main
git merge feature-branch

# 删除分支
git branch -d feature-branch
```

#### 4.4 远程仓库交互

```bash
# 查看远程仓库
git remote -v

# 添加远程仓库
git remote add origin https://github.com/username/repository.git

# 推送到远程仓库
git push -u origin main       # 首次推送
git push                      # 后续推送

# 拉取远程更新
git pull

# 获取远程更新但不合并
git fetch
```

### 5. 实用技巧

```bash
# 撤销工作区的修改
git checkout -- filename.txt

# 撤销暂存区的文件
git reset HEAD filename.txt

# 修改最后一次提交
git commit --amend

# 暂存当前修改
git stash

# 恢复暂存的修改
git stash pop
```

### 总结

- **Git 核心流程**: 修改文件 → `git add` → `git commit` → `git push`
- **分支管理**: 使用分支进行功能开发，完成后合并回主分支，保持代码整洁
- **协作基础**: 定期 `git pull` 获取最新代码，`git push` 分享你的贡献

掌握这些基础操作后，你就可以开始使用 Git 进行版本控制和团队协作了。随着使用深入，你可以进一步学习更高级的 Git 技巧，如变基 (rebase)、标签 (tag) 和交互式变基等。
